# SaaS Brasil Checklist de Cliente Piloto

## Objetivo

Este checklist ajuda a filtrar clientes para piloto pago assistido do SaaS Brasil V1. A prioridade e escolher clientes com dor real, escopo simples e disposicao para feedback.

## Checklist principal

- [ ] O cliente entende claramente o problema que quer resolver?
- [ ] O cliente tem rotina real de estoque, obra ou relatorio?
- [ ] O cliente aceita testar com acompanhamento?
- [ ] O cliente aceita reportar bugs e dificuldades?
- [ ] O cliente aceita pagar um valor inicial pelo piloto?
- [ ] O cliente tem celular Android para operacao de campo?
- [ ] O cliente precisa funcionar offline?
- [ ] O cliente precisa de multiplos funcionarios?
- [ ] O cliente tem risco de dados sensiveis?
- [ ] O cliente precisa emissao fiscal?

## Regra sobre emissao fiscal

Se o cliente precisa emissao fiscal, nao prometer agora. A V1 nao deve ser vendida como emissor de nota, ERP fiscal ou sistema contabil.

## Sinais positivos

- o cliente ja controla estoque ou obra em papel, planilha ou WhatsApp;
- o cliente perde tempo conferindo saldo;
- o cliente tem funcionario fazendo entrada e saida;
- o cliente precisa operar no celular;
- o cliente aceita comecar pequeno;
- o cliente entende que IA apoia, mas nao decide sozinha;
- o cliente aceita piloto de 15 a 30 dias.

## Sinais de alerta

- cliente exige ERP completo;
- cliente exige emissao de nota fiscal;
- cliente exige financeiro completo;
- cliente exige integracao complexa no piloto;
- cliente nao aceita reportar bugs;
- cliente nao aceita pagar;
- cliente tem dados sensiveis de saude sem contrato e compliance;
- cliente exige garantia de que a IA nunca erra;
- cliente quer CADISTA como projeto legal aprovado.

## Decisao recomendada

### Aceitar piloto

Aceitar quando o cliente tem dor real, escopo simples, aceita acompanhamento, aceita pagar e nao exige funcionalidades fora da V1.

### Aceitar com restricoes

Aceitar quando ha valor comercial, mas o cliente precisa de offline, multiplos funcionarios ou rotina mais intensa. Nesse caso, documentar limites antes do inicio.

### Nao aceitar agora

Nao aceitar quando o cliente exige nota fiscal, ERP completo, compliance de saude sem preparacao, projeto legal aprovado ou operacao em escala sem suporte.

## Registro minimo do piloto

Antes de iniciar, registrar:

- nome do cliente;
- segmento;
- oferta escolhida;
- dor principal;
- numero aproximado de usuarios;
- necessidade offline;
- existencia de dados sensiveis;
- expectativa de preco;
- prazo do piloto;
- criterios de sucesso.
